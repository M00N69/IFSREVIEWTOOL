Outil de Revue IFS

Solution complète pour la gestion du suivi des actions correctives suite aux audits IFS

Introduction à l'outil

L'outil de revue IFS est une solution innovante permettant de gérer efficacement les plans d'actions correctives suite aux audits IFS. Il facilite la collaboration entre les différents acteurs impliqués dans le processus.

Collaboration sans serveur

Échange de données via des packages compressés, permettant de travailler sans connexion internet permanente.

Accès sécurisé

Contrôle des droits d'accès en fonction du rôle (Auditeur, Site audité, Reviewer).

Traçabilité complète

Historique détaillé de toutes les actions et modifications effectuées.

Gestion des actions

Suivi détaillé de la mise en œuvre des corrections et actions correctives.

Notre outil se compose de trois interfaces web distinctes, chacune destinée à un acteur spécifique du processus d'audit IFS :

Point clé : Chaque interface est une application web indépendante avec ses propres fonctionnalités. Les trois applications partagent cependant une base de code commune et échangent des données via les packages .ifsaudit.

Contenu d'un package .ifsaudit :

  • Métadonnées : COID, nom du site, date d'audit, version...
  • Items d'audit : Liste des exigences, constats et actions
  • Commentaires : Échanges entre les acteurs (filtrés selon destinataire)
  • Preuves : Photos et documents attachés aux items
  • Logs : Historique complet des modifications

Les packages .ifsaudit sont des fichiers compressés contenant toutes les informations nécessaires au suivi du plan d'action. Ils peuvent être échangés par email ou tout autre moyen de transfert de fichiers.

Workflow complet

Voici les étapes détaillées du processus de gestion des actions correctives avec notre outil :

1

Import Excel par l'Auditeur

L'auditeur importe le fichier Excel contenant les exigences IFS et les constats de non-conformité identifiés lors de l'audit. Le système extrait les métadonnées (COID, site, type d'audit) et crée la structure de base du plan d'action.

2

Export du package pour le Site

L'auditeur exporte un package .ifsaudit contenant le plan d'action initial et l'envoie au site audité. Le statut du package est défini comme "SiteInputRequired" pour indiquer que le site doit compléter les informations.

3

Traitement par le Site

Le site audité charge le package reçu et complète pour chaque non-conformité :

  • Les corrections immédiates mises en place
  • Les actions correctives pour éviter la récurrence
  • Les responsables et échéances
  • Les preuves (photos, documents)

Le site exporte ensuite un nouveau package avec statut "AuditorReview".

4

Vérification par l'Auditeur

L'auditeur examine les corrections et preuves soumises par le site. Il peut :

  • Demander des compléments au site (retour à l'étape 3)
  • Mettre à jour les statuts des actions (Ouvert, En cours, Clôturé)
  • Ajouter des commentaires pour le site ou pour le reviewer
  • Exporter le package pour le reviewer avec statut "ReviewerReview"
5

Revue par le Reviewer

Le reviewer charge le package et :

  • Examine l'ensemble du plan d'action et des preuves
  • Ajoute des commentaires pour l'auditeur (invisibles au site)
  • Exporte le package avec statut "AuditorReview"
6

Finalisation par l'Auditeur

Après avoir intégré les retours du reviewer et les derniers échanges avec le site, l'auditeur finalise l'audit en définissant le statut à "Finalized". Le package final contient l'historique complet et ne peut plus être modifié.

Traçabilité complète

Tout au long du processus, le système enregistre automatiquement chaque action dans le composant logs du package, permettant de reconstituer l'historique complet de l'audit.

Structure du code

Fichiers principaux :

  • common.js : Fonctions utilitaires partagées entre les interfaces
  • auditeur.html / auditeur.js : Interface et logique pour l'auditeur
  • site.html / site.js : Interface et logique pour le site audité
  • reviewer.html / reviewer.js : Interface et logique pour le reviewer

Fonctions communes (common.js)

Le fichier common.js est central dans l'architecture. Il contient :

  • Compression et décompression : Utilisation de pako.js pour la gestion des packages
  • Gestion de fichiers : Import Excel, chargement/export de packages
  • Traçabilité : Enregistrement de toutes les actions dans les logs
  • Stockage local : Utilisation d'IndexedDB pour les données persistantes
  • Utilitaires UI : Fonctions de manipulation du DOM, messages d'erreur/info

async function compressJson(jsonData) {
    // Convertir l'objet en chaîne JSON
    const jsonString = JSON.stringify(jsonData);
    
    // Compresser avec pako.js
    const compressedBinary = pako.deflate(jsonString);
    
    // Encodage en Base64
    return new Promise((resolve, reject) => {
        const blob = new Blob([compressedBinary]);
        const reader = new FileReader();
        reader.onloadend = () => {
            const base64String = reader.result.split(',')[1];
            resolve(base64String);
        };
        reader.onerror = reject;
        reader.readAsDataURL(blob);
    });
}

Technologies utilisées

Frontend

HTML5, JavaScript et Tailwind CSS pour une interface moderne et réactive.

Stockage local

IndexedDB pour la persistance des données et le suivi de l'état de lecture.

Compression

Pako.js pour la compression et décompression des packages .ifsaudit.

Traitement Excel

SheetJS (XLSX) pour l'import et l'analyse des fichiers Excel.

Toutes ces technologies sont chargées via CDN, ne nécessitant aucune installation côté serveur ou client. L'application est entièrement autonome et peut fonctionner hors ligne une fois chargée.

Utilisation de l'interface Auditeur

Présentation

L'interface Auditeur est identifiable par sa barre de navigation bleue. C'est le point de départ du processus d'audit où vous pouvez importer un fichier Excel, créer un package, communiquer avec le Site et le Reviewer, et finaliser l'audit.

Premiers pas

  1. Saisissez votre nom dans le champ en haut à droite pour identifier vos actions dans les logs.
  2. Importez un fichier Excel en cliquant sur le bouton "Importer Excel". Ce fichier doit contenir les exigences et constats d'audit.
  3. Une fois le fichier importé, vous verrez apparaître la liste des éléments du plan d'action dans le tableau principal.

Fonctionnalités principales

  • Exporter un package : Générez un fichier .ifsaudit à envoyer au Site ou au Reviewer.
  • Charger un package : Ouvrez un fichier .ifsaudit reçu du Site ou du Reviewer.
  • Éditer un élément : Cliquez sur une ligne du tableau pour ouvrir la fenêtre d'édition.
  • Ajouter des commentaires : Dans la fenêtre d'édition, ajoutez des commentaires pour le Site ou pour le Reviewer.
  • Voir les preuves : Visualisez les documents et photos joints par le Site.
  • Finaliser l'audit : Une fois toutes les actions validées, finalisez le plan d'action.
Interface Auditeur avec le tableau des items d'audit

Utilisation de l'interface Site

Présentation

L'interface Site est identifiable par sa barre de navigation verte. Elle permet de répondre aux constats d'audit, de documenter les corrections et actions correctives, et d'ajouter des preuves de mise en œuvre.

Premiers pas

  1. Saisissez votre nom dans le champ en haut à droite.
  2. Chargez le package reçu de l'auditeur en cliquant sur "Charger Package".
  3. Le tableau principal affichera la liste des exigences et constats à traiter.

Fonctionnalités principales

  • Renseigner les actions : Documentez les corrections et actions correctives pour chaque item.
  • Ajouter des preuves : Joignez des photos ou documents justificatifs (PDF, images).
  • Répondre aux commentaires : Lisez les commentaires de l'auditeur et répondez si nécessaire.
  • Exporter le package : Générez un fichier .ifsaudit à renvoyer à l'auditeur.
Interface Site avec la fenêtre d'édition d'un item

Utilisation de l'interface Reviewer

Présentation

L'interface Reviewer est identifiable par sa barre de navigation violette. Elle permet d'examiner l'ensemble du plan d'action, d'accéder à toutes les preuves et de communiquer directement avec l'auditeur.

Premiers pas

  1. Saisissez votre nom dans le champ en haut à droite.
  2. Chargez le package reçu de l'auditeur en cliquant sur "Charger Package".
  3. Le tableau principal affichera la liste complète des exigences, constats et réponses du site.

Fonctionnalités principales

  • Consulter les détails : Examinez les informations complètes pour chaque item d'audit.
  • Voir les preuves : Visualisez tous les documents joints par le site.
  • Ajouter des commentaires : Rédigez des commentaires à l'attention de l'auditeur (invisibles au site).
  • Consulter l'historique : Accédez à la traçabilité complète des modifications.
  • Exporter le package : Générez un fichier .ifsaudit avec vos commentaires pour l'auditeur.
